GPS Attendance System

Second Year Project - Location-Based Attendance Tracker

Project Overview

A GPS-based attendance tracking system developed during my second year that uses geolocation technology to verify student presence at specific locations. This system ensures accurate attendance by validating that students are physically present at the designated venue.

Key Features

  • GPS location tracking and verification
  • Geofencing with radius validation
  • Real-time location detection
  • Student database management
  • Attendance history with location data
  • User authentication system

Technologies Used

HTML CSS Java EE 9 JSP Servlets MySQL JavaScript Apache Tomcat GPS API

Project Purpose

This GPS-based attendance system was created to address the issue of proxy attendance and ensure that students are physically present at the event or class location. By leveraging geolocation technology, the system verifies that check-ins occur only within a specified radius of the venue, providing authentic attendance records.

How GPS Tracking Works

  • System captures student's current GPS coordinates when marking attendance
  • Compares location against predefined event/class coordinates
  • Calculates distance using geographical distance formula
  • Allows attendance only if student is within specified radius (e.g., 100 meters)
  • Prevents attendance marking from outside the designated area

Core Functionality

For Teachers/Administrators

  • Set venue coordinates for events or classes
  • Configure geofence radius for location validation
  • Monitor real-time attendance with location data
  • View attendance records with GPS coordinates
  • Generate location-verified attendance reports

For Students

  • Mark attendance from mobile devices with GPS enabled
  • System automatically detects current location
  • Receive confirmation when successfully checked in
  • Get alerts if outside the allowed radius
  • View personal attendance history with timestamps

Technical Implementation

The system uses the browser's Geolocation API to capture the user's current GPS coordinates. When a student attempts to mark attendance, JavaScript retrieves their latitude and longitude, then sends this data to the Java Servlets backend. The server calculates the distance between the student's location and the venue using the Haversine formula, which accounts for the Earth's curvature. If the calculated distance is within the allowed radius, the attendance is recorded in the MySQL database along with the GPS coordinates and timestamp. The application runs on Apache Tomcat and uses JSP for dynamic page rendering.

Security Features

  • GPS spoofing detection mechanisms
  • Secure user authentication and session management
  • Time-based validation to prevent backdated entries
  • Encrypted storage of location data

Learning Outcomes

This project provided hands-on experience with geolocation technologies, spatial calculations, and location-based services. I learned how to work with GPS APIs, implement distance calculation algorithms, handle location permissions, and create mobile-responsive interfaces for location-based applications.

Challenges and Solutions

  • GPS Accuracy: Implemented averaging of multiple readings to improve accuracy
  • Battery Consumption: Optimized to request location only when needed
  • Privacy Concerns: Store only necessary location data with user consent

Future Improvements

  • Visual map display showing venue and student locations
  • Analytics dashboard for attendance patterns
