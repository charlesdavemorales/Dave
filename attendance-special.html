System Purpose

The RFID University Attendance Management System provides a fast, accurate, and comprehensive solution for tracking student participation in campus events. Using contactless RFID technology, students can check in and out instantly, while the system maintains detailed records of attendance, calculates fines for missed events, and provides online access to all attendance data.

Production Deployment: This system is currently being used by the university to manage attendance for mandatory events, assemblies, and activities. It has successfully processed thousands of RFID scans with high accuracy and reliability, serving the entire student body and faculty.

RFID Technology Integration

The system uses RFID (Radio-Frequency Identification) technology for seamless, contactless attendance tracking:

  • Each student is issued a unique RFID card or tag linked to their account
  • RFID readers installed at event venues automatically detect and scan cards
  • Check-in/check-out occurs in less than a second with a simple card tap
  • Each RFID tag has a unique identifier preventing duplication
  • System instantly records timestamp and event details in the database

System Architecture

Built with modern PHP and web technologies for robust performance:

  • Backend: PHP server-side processing for RFID data handling and business logic
  • Database: MySQL with optimized schema for attendance, fines, and time tracking
  • Frontend: Responsive HTML, CSS, and JavaScript for online access portal
  • Hardware Integration: RFID readers connected via serial/USB interface with PHP serial communication

Advanced Features

Automated Fine System

The system automatically identifies students who miss mandatory events and calculates fines based on configurable rules. Administrators can set different fine amounts for different event categories, apply grace periods, and track payment status. Students receive automated email notifications about pending fines and can view detailed breakdowns online.

Comprehensive Time Tracking

Every RFID tap is recorded with precise timestamps. The system tracks both check-in and check-out times, calculates duration of stay, and validates whether students met minimum attendance requirements. This data is crucial for events that require students to stay for the entire duration.

Online Access Portal

Students can log in from anywhere to view their complete attendance history, upcoming events, fine status, and personal statistics. The portal provides transparency and allows students to plan ahead for mandatory events and manage their attendance requirements proactively.

Security Features

  • Complete audit trail of all system activities
  • Card deactivation for lost or stolen RFID tags

Technical Implementation

RFID Integration: The system interfaces with RFID readers through serial communication using PHP's serial port libraries. When a card is scanned, the reader sends the unique tag ID to the PHP application, which queries the MySQL database to identify the student and record attendance with timestamp.

Database Schema: Normalized relational database with tables for students, events, RFID cards, attendance records (with time_in and time_out fields), fines, and payment history. Implements foreign key constraints and indexes for optimal query performance.

Key Technologies:

  • PHP Serial: Serial port communication for RFID reader connectivity
  • FPDF/TCPDF: PDF generation for attendance reports and receipts
  • PHPMailer: Email notifications for events and fines
  • MySQLi/PDO: Secure database connectivity and prepared statements
  • AJAX/Fetch API: Real-time updates without page refresh
