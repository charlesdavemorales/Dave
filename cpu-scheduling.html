Preemptive CPU Scheduling Algorithm Simulator

Third Year Project - Operating Systems

Project Overview

An interactive simulator that implements and visualizes preemptive CPU scheduling algorithms used in modern operating systems. The program demonstrates how the operating system can interrupt running processes to allocate CPU time more efficiently based on priority and time constraints.

Implemented Preemptive Algorithms

  • Round Robin (RR) - Time quantum based preemption
  • Shortest Remaining Time First (SRTF) - Preemptive SJF
  • Priority Scheduling (Preemptive) - Higher priority can interrupt
  • Multilevel Queue Scheduling - Multiple ready queues

Technologies Used

Python Algorithm Design Data Structures Visualization

Features & Functionality

The simulator focuses on preemptive scheduling where the operating system can interrupt a running process to allocate the CPU to another process. It calculates key performance metrics including average waiting time, turnaround time, response time, and context switch overhead - which is particularly important in preemptive systems.

Users can input custom process sets with arrival times, burst times, and priorities. The program generates detailed Gantt charts showing process preemptions and context switches, making it easy to visualize how processes are interrupted and resumed during execution.

Performance Metrics

  • Average Waiting Time
  • Average Turnaround Time
  • Average Response Time
  • Context Switch Count & Overhead

Learning Outcomes

This project deepened my understanding of preemptive scheduling mechanisms in modern operating systems. I gained insights into how context switching affects system performance, the trade-offs between responsiveness and overhead, and how different preemptive algorithms optimize for various metrics like response time and fairness in multitasking environments.
